## 🔒 ルール①：H2並び順の鉄則【全記事必須】

**Why:** 2026-05-06 リベ大添削で記事20に「ペルソナがカメラを売りたい人なら、読者が知りたいことを最初に持ってきた方が読者ファーストになる（相場・高く売る方法・売る先）」と指摘。

**How to apply:** 全記事のH2並び順は以下を必須とする。

### 必須H2並び順（売却系・キラー記事の標準）
1. **H2-1：相場（いくら？）** — 読者が真っ先に知りたい数字を冒頭で出す
2. **H2-2：高く売るコツ（どう売る？）** ＋ 末尾にミニCTA（上CTA）
3. **H2-3：売り先の選択肢（どこに売る？）** — 5択など並列で提示
4. **H2-4：おすすめ業者（mybest風フルカード）** ＋ フルA8 CTA（中CTA）
5. **H2-5：状態別の対応**（カビ・故障・ジャンク・古いetc）
6. **H2-6〜：業界トレンド・歴史・市場動向**（深掘りしたい人向け・後半）
7. **H2-N：まとめ** ＋ 下CTA（下A8カード）

### 配置ルール
- **冒頭4本 = 行動に直結する情報**（相場→売り方→売り先→業者）
- **後半 = トレンド・歴史・業界解説**
- **リード文末に1行誘導**：「※業界トレンド・市場動向は記事後半で詳しく」
- **A8 CTAは「上（H2-2末尾ミニ）／中（H2-4業者カード）／下（まとめ前）」の3点配置堅守**
- 内部リンクカード3枚は記事末尾（まとめ後）に固める

### 🚫 NG構成
- H2-1で「ブーム終了？」「時代遅れ？」など業界トレンド解説から入る → 読者離脱

### Phase 2チェック項目

- [ ] 業界トレンド・歴史解説が記事後半（H2-5以降）に配置

## 🔒 ルール②：1記事1焦点の鉄則【全記事必須】

**Why:** 2026-05-06 リベ大添削で記事20に「ブーム（世間の注目度）の話と高く売る話が混在していて、話の焦点が分かりづらい」と指摘。1記事に2つのテーマを併走させると読者離脱。お店で例えると「カメラ買取コーナー」と「カメラ博物館」の同居状態＝集中して買い物できない。

**How to apply:**
- **メインテーマは「読者が何を解決したいか」1つに絞る**
- **メインテーマと違う話題（業界トレンド・歴史・市場動向）は最大1H2のコラムに圧縮**
- **「Aの話とBの話」が並行する構造はNG** — 末尾に「【業界人コラム】」型でB話題を1本まとめる
- **タイトルKWに業界トレンド系（ブーム終了・時代遅れ等）を入れた場合**：本文メインは「売る話」、トレンドは末尾コラムで結論統合

### 焦点ぼやけの典型パターン
| 散漫構造（NG） | 焦点絞り（OK） |
|---|---|
| H2-1「ブーム終了の理由」H2-2「ブームではない根拠」H2-3「相場」H2-4「売り方」… | H2-1「相場」H2-2「売り方」…H2-N「【業界人コラム】ブーム終了は誤解」 |

### 違反履歴
- **2026-05-06 記事20**：旧構造でH2-1（終了理由）H2-2（終了根拠）が冒頭2本→ブーム関連2H2を【業界人コラム】1本に統合

## 🔒 装飾統一ルール（既存・🔒変更禁止）

### 🚨 HTMLブロック内の太字記法（2026-05-14 Satoshi指摘）

**Why:** HTMLブロック内（`<div>...</div>` `<p>...</p>` 等の中）でMarkdownの `**強調**` 記法を使うと、Astro が処理せず `**` がそのまま画面に表示される（記事33で発覚）。

**How to apply:**
- HTMLブロック内テキスト：**必ず `<strong>...</strong>` を使う**
- 純Markdown段落（HTMLタグで囲まれてない部分）：`**...**` でもOK
- 迷ったら `<strong>` で統一が安全

```html
<!-- ❌ NG（** が生表示される） -->
<div>説明文 **強調したい部分** です。</div>

<!-- ✅ OK -->
<div>説明文 <strong>強調したい部分</strong> です。</div>
```

## 🆕 ルール⑨：視覚要素 最低1個 必須【2026-05-21 Satoshi確定】

### 鉄則：全記事に最低1個、視覚的にわかりやすい要素を必ず入れる

**Why:** 2026-05-21、毛皮集客記事Phase 1完了時にSatoshi「これから記事に1個は表とかイラストとか視覚的にわかりやすいものも入れていきたい」と明言。朝のサイト分析レポート（デザイン採点 6.5/10）でも「本文中盤に比較表・要点サマリーを配置すると滞在時間・回遊率上がる」と指摘済。テキストばかりの記事はスマホ流し読み層に弱い・離脱率が高い。

**How to apply:** Phase 2執筆時、必ず以下のいずれか1個以上を本文中盤（H2-2 or H2-3 あたり）に配置する。複数入れるのが理想。

### ✅ 採用可能な視覚要素 6種

| 種類 | 用途 | 強み |
|---|---|---|
| **①比較表（HTML table）** | 選択肢メリデメ・業者比較・状態別相場 | SEO強化・自分事化・滞在時間UP |
| **②チェックリストBOX** | ミニ診断・該当条件チェック | 読者の判断補助・CV誘導 |
| **③インフォグラフィック画像** | フロー・手順・関係図 | 視覚的理解・SNS拡散 |
| **④イラスト/写真** | 体験談シーン・現場ビジュアル | 共感・信頼感 |
| **⑤フローチャート** | 判断分岐・Yes/No 選択 | 行動誘導 |
| **⑥アイコン入りBOX** | 注意点3つ・対策ステップ | 視認性・要点整理 |

### 📐 配置パターン推奨

| 記事タイプ | 推奨視覚要素 |
|---|---|
| **キラー記事**（業者評判・比較・買取系） | ①比較表（業者×報酬×特徴）＋④査定現場写真 |
| **集客記事**（処分・後悔・もったいない系） | ①比較表（選択肢×メリデメ）＋②ミニ診断 |
