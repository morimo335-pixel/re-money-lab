# 🚫 記事間 使い回し禁止フレーズリスト（2026-06-12 確定）

**目的**：1記事内のAI臭は消せても、記事をまたいで署名フレーズが複製されると、連続で読む読者とサイト全体を評価するGoogleに「量産テンプレ」と見える。73の口コミ重複検知と同じ仕組みの"文章版"。

## 運用ルール（執筆時・公開前に必ず）
新記事を書いたら公開前に下記を実行し、🚨が出たフレーズは**その記事だけ別表現に書き換える**（既存記事は凍結中のため触らない・自然に置き換わるのを待つ）。

```bash
cd ~/Desktop/ClaudeCode作業/re-money-lab-astro
for p in "見てきました" "見送ってきました" "立ち会ってきました" "目にしてきました" "もったいない手放し方" "取りこぼし" "いちばん失敗が少ない第一歩" "捨てるのが一番の損" "思わぬ高値になることもあります" "次の誰かの" "持ち主が価値を見誤りやすい" "まず希望のある話から" "まず、希望のある話から" "命をつなぐ" "報われる" "現場ではよくあること"; do
  n=$(grep -l "$p" src/content/blog/*.md 2>/dev/null | wc -l | tr -d ' ')
  [ "$n" -ge 2 ] && echo "🚨[$n本] $p" || echo "  [$n本] $p"
done
# 🆕 2026-06-13 強化：「見てきました」は語尾レベルで検出（同義置換の逃げを塞ぐ）。1本でもヒットしたら構造ごと書き換える。
```
